# pi-tps

### Tokens-per-second tracker for pi

**[Install](#install)** · **[How it works](#how-it-works)**

See your LLM generation speed (tokens/second) and TTFT (time to first token) after every agent turn.

## What's included

|               |                                                         |
| ------------- | ------------------------------------------------------- |
| **Extension** | Tracks token usage, TTFT, and TPS after each agent turn |

### Features

- **TTFT tracking**: Measures time from request sent to first token received
- **Accurate TPS**: Excludes network latency, measures only token generation time
- **Token aggregation**: Sums up all assistant message tokens (input, output)
- **Clean output format**: `TPS X.X tok/s · TTFT X.Xs · X.Xs · out X · in X`
- **Notification banner**: Shows stats in a transient popup notification
- **Persisted notifications**: Notifications are restored on session resume

## How it works

The extension hooks into pi's lifecycle events to provide accurate timing:

1. **`turn_start`**: Captures when the request is sent to the LLM
2. **`message_start`**: Captures when the first token arrives (TTFT)
3. **`message_end`**: Updates when each assistant message completes
4. **`agent_end`**: Calculates stats and shows notification

### Timing breakdown

| Phase          | Measured by                                |
| -------------- | ------------------------------------------ |
| **TTFT**       | `turn_start` → first `message_start`       |
| **Generation** | first `message_start` → last `message_end` |
| **Total**      | `turn_start` → last `message_end`          |

This approach excludes:

- Network latency (included in TTFT)
- Tool execution time between turns
- Server queue time (included in TTFT)

### Output format

```
TPS 42.5 tok/s · TTFT 1.2s · 29.7s · out 1,234 · in 567
```

| Field  | Description                             |
| ------ | --------------------------------------- |
| `TPS`  | Tokens per second (output / generation) |
| `TTFT` | Time to first token in seconds          |
| `s`    | Total time from request to completion   |
| `out`  | Output tokens generated by the LLM      |
| `in`   | Input tokens sent to the LLM            |

The stats are saved as a custom entry (type `tps`) which persists in the session and is restored as a notification when you resume the session.
